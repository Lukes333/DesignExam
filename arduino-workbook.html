LESSON 1: The Anatomy of Arduino Code

Module 1: Foundation ‚òÖ‚òÜ‚òÜ Easy ‚è± 20 minutes

YOU WILL LEARN:

  • What is setup() and why we need it
  • What is loop() and how it works
  • Why these two functions are mandatory
  • Basic code structure rules

CONCEPT

Every Arduino program (called a "sketch") has TWO main parts:

  1. setup() - Runs ONCE when Arduino powers on
    • Used to configure pins
    • Start communication
    • Initialize variables
    • Set up components
  2. loop() - Runs FOREVER after setup()
    • Main program logic
    • Reads sensors
    • Controls outputs
    • Repeats automatically

Think of it like this:

‚Ä¢ setup() = Getting dressed in the morning (once)

‚Ä¢ loop() = Breathing (continuous, automatic)

DETAILED EXPLANATION

// ===== OUTSIDE ALL FUNCTIONS =====
// This area is for DECLARATIONS
// - Pin numbers
// - Global variables
// - Library includes

const int ledPin = 13;  // DECLARE a constant (won't change)

// ===== SETUP FUNCTION =====
void setup() {
  // This runs EXACTLY ONCE when Arduino starts

  pinMode(ledPin, OUTPUT);  // CONFIGURE the pin

  // Why in setup()?
  // Because we only need to configure pins once!
}

// ===== LOOP FUNCTION =====
void loop() {
  // This runs FOREVER, over and over

  digitalWrite(ledPin, HIGH);  // Turn LED on
  delay(1000);                 // Wait 1 second
  digitalWrite(ledPin, LOW);   // Turn LED off
  delay(1000);                 // Wait 1 second

  // Then loop() automatically starts again!
  // So LED blinks forever
}
WHY THIS STRUCTURE?

‚Ä¢ Arduino is designed to run ONE program continuously

‚Ä¢ Setup ensures hardware is ready before main program

‚Ä¢ Loop allows automatic repetition without complex code

‚Ä¢ This pattern is UNIVERSAL - every Arduino program uses it

CRITICAL RULES:

‚úÖ MUST have both setup() and loop()

‚úÖ setup() comes before loop() in your code

‚úÖ Both use lowercase (Setup() won't work!)

‚úÖ Both have void before them

‚úÖ Both have empty parentheses: ()

‚úÖ Both have curly braces: { }

COMMON MISTAKES

‚úÖ Correct:

‚úÖ Correct:

QUICK QUIZ

2. How many times does loop() run?

ANSWERS:

1. a) Once - when Arduino powers on or resets

3. b) setup() then loop() - standard convention for readability

KEY TAKEAWAYS

  • Every Arduino sketch has setup() and loop()
  • setup() runs once at startup - use for initialization
  • loop() runs forever - use for main program logic
  • Both functions must be lowercase and start with void
  • This structure is the foundation of ALL Arduino programs

LESSON 2: Variables and Constants

Module 1: Foundation ‚òÖ‚òÜ‚òÜ Easy ‚è± 25 minutes

YOU WILL LEARN:

  • Difference between const and regular variables
  • Where to declare variables (global vs local)
  • Naming rules and best practices
  • Data types (int, float, bool, long)

CONCEPT

Variables are containers that hold values. Like labeled boxes.

// CONSTANTS (values that NEVER change)
const int ledPin = 13;        // Pin numbers - never change
const int threshold = 500;    // Fixed threshold - never change

// VARIABLES (values that CAN change)
int sensorValue = 0;          // Sensor reading - changes constantly
bool ledState = false;        // LED on/off - toggles
float distance = 0.0;         // Distance measurement - changes

WHY const vs regular variable?

‚Ä¢ const = PROTECTION - Arduino won't let you accidentally change it

‚Ä¢ const = CLARITY - tells you "this value is fixed"

‚Ä¢ const = MEMORY - Arduino can optimize better

DATA TYPES

INT (whole numbers: -32,768 to 32,767)

int ledPin = 13;           // Pin numbers
int sensorValue = 512;     // Analog reading (0-1023)
int angle = 90;            // Servo angle (0-180)

FLOAT (decimal numbers)

float distance = 25.3;     // Distance in cm
float voltage = 3.7;       // Voltage calculation
float temperature = 23.5;  // Temperature in ¬∞C

BOOL (true/false only)

bool ledState = true;           // LED on or off
bool motionDetected = false;   // Sensor triggered?

LONG (very large whole numbers)

long duration = 1234567;   // Ultrasonic timing (microseconds)
NAMING RULES:

‚úÖ Start with lowercase letter

‚úÖ Use camelCase for multiple words: greenLedPin

‚úÖ Be descriptive: distanceThreshold not d

‚úÖ No spaces: ledPin not led pin

‚ùå Don't use reserved words: int, void, loop

WHERE TO DECLARE

// ===== GLOBAL (top of file - accessible everywhere) =====
const int ledPin = 13;     // Accessible in setup() and loop()

void setup() {
  // ===== LOCAL (inside function - only here) =====
  int i = 0;               // Only exists in setup()

  pinMode(ledPin, OUTPUT); // Can use global variable
}

void loop() {
  // int i;                // ERROR if you try to use i from setup()

  int sensorValue = 0;     // Local to loop() only
}

RULE: Declare pin numbers and constants globally (top). Declare temporary values locally (inside functions).

KEY TAKEAWAYS

  • Use const for values that never change (pin numbers, thresholds)

  • Use descriptive names in camelCase
  • No spaces in variable names - use camelCase or underscores

LESSON 3: Pin Modes and Digital I/O

Module 1: Foundation ‚òÖ‚òÖ‚òÜ Medium ‚è± 30 minutes

YOU WILL LEARN:

  • What pinMode() does and why it's necessary
  • INPUT vs OUTPUT vs INPUT_PULLUP modes
  • digitalWrite() for controlling outputs
  • digitalRead() for reading inputs
  • When to use each mode

CONCEPT

Arduino pins are like doors. You must tell them if things go IN or OUT.

pinMode(pin, MODE);

// Three modes:
// OUTPUT - Arduino SENDS signal (controls LED, motor, etc.)
// INPUT - Arduino RECEIVES signal (reads button, sensor, etc.)
// INPUT_PULLUP - Arduino RECEIVES signal with internal resistor

CRITICAL RULE:

‚Ä¢ If Arduino MAKES something happen ‚Üí OUTPUT

‚Ä¢ If Arduino CHECKS something ‚Üí INPUT or INPUT_PULLUP

DETAILED EXPLANATION

const int ledPin = 13;      // Arduino CONTROLS this
const int buttonPin = 2;    // Arduino READS this

void setup() {
  // LED: Arduino sends power ‚Üí OUTPUT
  pinMode(ledPin, OUTPUT);

  // Button: Arduino receives signal ‚Üí INPUT
  pinMode(buttonPin, INPUT_PULLUP);  // With internal resistor
}

void loop() {
  // READ button (INPUT pin)
  int buttonState = digitalRead(buttonPin);
  // Returns: HIGH or LOW

  // WRITE to LED (OUTPUT pin)
  digitalWrite(ledPin, HIGH);  // Turn on (5V)
  digitalWrite(ledPin, LOW);   // Turn off (0V)
}

WHEN TO USE EACH MODE:

// OUTPUT - When Arduino CONTROLS something
pinMode(ledPin, OUTPUT);        // LED - Arduino turns it on/off
pinMode(motorPin, OUTPUT);      // Motor - Arduino controls it
pinMode(buzzerPin, OUTPUT);     // Buzzer - Arduino makes it beep

// INPUT - When Arduino READS something (with external resistor)
pinMode(sensorPin, INPUT);      // Sensor with resistor circuit

// INPUT_PULLUP - When Arduino READS something (no external resistor needed)
pinMode(buttonPin, INPUT_PULLUP);  // Button to GND (no resistor!)

digitalWrite() - Control OUTPUT pins

digitalWrite(pin, value);

// Only two values:
digitalWrite(ledPin, HIGH);  // Turn ON (5V to pin)
digitalWrite(ledPin, LOW);   // Turn OFF (0V to pin)

digitalRead() - Read INPUT pins

int value = digitalRead(pin);

// Returns one of two values:
// HIGH (5V detected) or LOW (0V detected)

int buttonState = digitalRead(buttonPin);

if (buttonState == HIGH) {
  // Button pressed (if using INPUT_PULLUP, LOW = pressed!)
}
INPUT_PULLUP BEHAVIOR (IMPORTANT!):
pinMode(buttonPin, INPUT_PULLUP);  // Internal resistor pulls pin HIGH

// When button NOT pressed:
digitalRead(buttonPin);  // Returns HIGH

// When button pressed (connects to GND):
digitalRead(buttonPin);  // Returns LOW

// NOTE: It's REVERSED!
// LOW = pressed
// HIGH = not pressed

EXERCISE 3.2: Fix the Button Logic

This code should turn LED on when button pressed. What's wrong?

const int ledPin = 13;
const int buttonPin = 2;

void setup() {
  pinMode(ledPin, OUTPUT);
  pinMode(buttonPin, INPUT_PULLUP);
}

void loop() {
  int buttonState = digitalRead(buttonPin);

  if (buttonState == HIGH) {  // If button pressed...
    digitalWrite(ledPin, HIGH);  // Turn on LED
  } else {
    digitalWrite(ledPin, LOW);
  }
}

SOLUTION:

THE BUG: INPUT_PULLUP is REVERSED!

‚Ä¢ When button NOT pressed = HIGH

‚Ä¢ When button pressed = LOW

CORRECTED CODE:

const int ledPin = 13;
const int buttonPin = 2;

void setup() {
  pinMode(ledPin, OUTPUT);
  pinMode(buttonPin, INPUT_PULLUP);
}

void loop() {
  int buttonState = digitalRead(buttonPin);

  // INPUT_PULLUP: LOW = pressed, HIGH = not pressed
  if (buttonState == LOW) {  // Changed to LOW!
    digitalWrite(ledPin, HIGH);
  } else {
    digitalWrite(ledPin, LOW);
  }
}

WHY THIS HAPPENS:

INPUT_PULLUP internally connects pin to 5V through resistor. When button pressed, it connects to GND (0V), pulling pin LOW. This is opposite of what seems intuitive!

COMMON MISTAKES

‚ùå Using OUTPUT for button: pinMode(buttonPin, OUTPUT);

‚úÖ Correct: pinMode(buttonPin, INPUT_PULLUP);

‚ùå Forgetting INPUT_PULLUP is reversed:

‚úÖ Correct:

if (buttonState == LOW) // Correct! LOW = pressed with INPUT_PULLUP

‚ùå Using digitalRead() on OUTPUT pin

‚úÖ Correct: digitalRead() only for INPUT/INPUT_PULLUP pins

QUICK QUIZ

1. If Arduino CONTROLS an LED, what mode?

2. If Arduino READS a button (no resistor), what mode?

3. With INPUT_PULLUP, button pressed returns:

ANSWERS:

1. b) OUTPUT - Arduino sends signal to LED

2. c) INPUT_PULLUP - reads AND provides internal resistor

3. b) LOW - because button connects pin to GND

KEY TAKEAWAYS

  • Always call pinMode() in setup() before using a pin
  • OUTPUT = Arduino controls it | INPUT/INPUT_PULLUP = Arduino reads it
  • digitalWrite() for OUTPUT pins (HIGH/LOW)
  • digitalRead() for INPUT pins (returns HIGH/LOW)
  • INPUT_PULLUP is REVERSED: LOW = pressed, HIGH = not pressed

LESSON 4: Analog Input (analogRead)

Module 1: Foundation ‚òÖ‚òÖ‚òÜ Medium ‚è± 25 minutes

YOU WILL LEARN:

  • What analog input is and how it differs from digital
  • How to use analogRead()
  • Understanding the 0-1023 range (10-bit ADC)
  • Analog pins (A0-A5) on Arduino Uno
  • Common analog sensors (potentiometer, LDR, etc.)

CONCEPT

Digital vs Analog:

  • Digital: Only 2 states ‚Üí HIGH (5V) or LOW (0V)
  • Analog: Range of values ‚Üí 0V to 5V (any voltage in between)

analogRead() converts voltage to a number:

  • 0V ‚Üí returns 0
  • 2.5V ‚Üí returns 512
  • 5V ‚Üí returns 1023

Remember:

‚Ä¢ Digital pins: Read with digitalRead() ‚Üí returns HIGH or LOW

‚Ä¢ Analog pins: Read with analogRead() ‚Üí returns 0-1023

DETAILED EXPLANATION

// Analog input from potentiometer
const int potPin = A0;  // Analog pin A0
int potValue = 0;       // Variable to store the reading

void setup() {
  // No pinMode needed for analog INPUT!
  // Analog pins default to input
  Serial.begin(9600);   // Start serial for monitoring
}

void loop() {
  // Read analog value (0-1023)
  potValue = analogRead(potPin);

  // Display value
  Serial.print("Pot value: ");
  Serial.println(potValue);

  delay(100);  // Small delay for readability
}
KEY POINTS:

‚Ä¢ Analog pins: A0, A1, A2, A3, A4, A5 (6 pins on Arduino Uno)

‚Ä¢ Range: 0-1023 (10-bit resolution = 2^10 = 1024 values)

‚Ä¢ No pinMode() needed for analog input (defaults to INPUT)

‚Ä¢ Returns INT value, not voltage

Converting to Voltage:

int sensorValue = analogRead(A0);  // 0-1023
float voltage = sensorValue * (5.0 / 1023.0);  // Convert to 0-5V

// Example: if sensorValue = 512
// voltage = 512 * (5.0 / 1023.0) = 2.5V

LESSON 5: PWM Output (analogWrite)

Module 1: Foundation ‚òÖ‚òÖ‚òÜ Medium ‚è± 30 minutes

YOU WILL LEARN:

  • What PWM (Pulse Width Modulation) is and how it works
  • Which pins support PWM on Arduino Uno
  • How to use analogWrite()
  • Controlling LED brightness and motor speed
  • Understanding the 0-255 range

CONCEPT

PWM (Pulse Width Modulation) simulates analog output using rapid on/off switching.

Digital pins can only be HIGH (5V) or LOW (0V). But with PWM, we can simulate voltages in between by rapidly switching on and off.

How PWM Works:

‚Ä¢ analogWrite(pin, 0) = Always OFF = 0V = 0% duty cycle

‚Ä¢ analogWrite(pin, 64) = 25% ON, 75% OFF = ~1.25V

‚Ä¢ analogWrite(pin, 128) = 50% ON, 50% OFF = ~2.5V

‚Ä¢ analogWrite(pin, 191) = 75% ON, 25% OFF = ~3.75V

‚Ä¢ analogWrite(pin, 255) = Always ON = 5V = 100% duty cycle

PWM PINS ONLY!

Only these 6 pins support PWM on Arduino Uno (marked with ~ symbol):

3, 5, 6, 9, 10, 11

Using analogWrite() on other pins won't work!

DETAILED EXPLANATION

// PWM LED brightness control
const int ledPin = 9;  // MUST be PWM pin (3, 5, 6, 9, 10, 11)

void setup() {
  pinMode(ledPin, OUTPUT);  // Still need pinMode!
}

void loop() {
  // Fade in
  for (int brightness = 0; brightness <= 255; brightness++) {
    analogWrite(ledPin, brightness);
    delay(10);
  }

  // Fade out
  for (int brightness = 255; brightness >= 0; brightness--) {
    analogWrite(ledPin, brightness);
    delay(10);
  }
}

Different Brightness Levels:

analogWrite(ledPin, 0);    // OFF (0%)
analogWrite(ledPin, 64);   // 25% brightness
analogWrite(ledPin, 128);  // 50% brightness
analogWrite(ledPin, 191);  // 75% brightness
analogWrite(ledPin, 255);  // Full brightness (100%)
PWM vs Digital:

‚Ä¢ digitalWrite(pin, HIGH) = Always 5V

‚Ä¢ digitalWrite(pin, LOW) = Always 0V

‚Ä¢ analogWrite(pin, value) = Simulated 0V to 5V based on value (0-255)

  • analogWrite(pin, value) where value = 0-255

  • Use for: LED brightness, motor speed, any variable output

LESSON 6: Delay Functions

Module 2: Timing & Control ‚òÖ‚òÜ‚òÜ Easy ‚è± 20 minutes

YOU WILL LEARN:

  • How delay() works
  • Difference between milliseconds and microseconds
  • When to use delayMicroseconds()
  • Why delay() blocks your program
  • Time conversion (seconds ‚Üî milliseconds)

CONCEPT

delay() pauses your program for a specified time in milliseconds.

delayMicroseconds() pauses for microseconds (much shorter!).

Time Conversions:

‚Ä¢ 1 second = 1,000 milliseconds (ms)

‚Ä¢ 1 millisecond = 1,000 microseconds (Œºs)

‚Ä¢ 1 second = 1,000,000 microseconds

Examples:

‚Ä¢ delay(1000) = 1 second

‚Ä¢ delay(500) = 0.5 seconds

‚Ä¢ delayMicroseconds(10) = 0.00001 seconds

DETAILED EXPLANATION

void loop() {
  digitalWrite(ledPin, HIGH);  // Turn LED on
  delay(1000);                 // Wait 1 second (1000 ms)
  digitalWrite(ledPin, LOW);   // Turn LED off
  delay(500);                  // Wait 0.5 seconds (500 ms)
  // LED blinks: ON for 1 sec, OFF for 0.5 sec
}
IMPORTANT: delay() BLOCKS!

During delay(), Arduino does NOTHING else!

CONCEPT

if statements let Arduino make decisions based on conditions.

Think of it like:

ELSE ‚Üí leave umbrella at home

DETAILED EXPLANATION

Basic If/Else:

int buttonState = digitalRead(buttonPin);

if (buttonState == LOW) {  // If pressed (INPUT_PULLUP)
  digitalWrite(ledPin, HIGH);  // Turn on LED
} else {  // If not pressed
  digitalWrite(ledPin, LOW);   // Turn off LED
}

Multiple Conditions (if/else if/else):

int distance = getDistance();

if (distance < 10) {
  // Very close - red LED
  digitalWrite(redLed, HIGH);
  digitalWrite(yellowLed, LOW);
  digitalWrite(greenLed, LOW);
} else if (distance < 30) {
  // Medium distance - yellow LED
  digitalWrite(redLed, LOW);
  digitalWrite(yellowLed, HIGH);
  digitalWrite(greenLed, LOW);
} else {
  // Far away - green LED
  digitalWrite(redLed, LOW);
  digitalWrite(yellowLed, LOW);
  digitalWrite(greenLed, HIGH);
}
How It Works:

1. Checks first condition (distance < 10)

2. If TRUE, runs that block and SKIPS the rest

3. If FALSE, checks next condition (distance < 30)

4. If all conditions are FALSE, runs else block

LESSON 8: Comparison Operators

Module 2: Timing & Control ‚òÖ‚òÜ‚òÜ Easy ‚è± 20 minutes

YOU WILL LEARN:

  • All comparison operators: ==, !=, <, >, <=, >=
  • How to use them in if statements
  • Testing ranges of values
  • Avoiding common mistakes

CONCEPT

Comparison operators compare two values and return TRUE or FALSE.

The 6 Comparison Operators:

‚Ä¢ == Equal to

‚Ä¢ != Not equal to

‚Ä¢ < Less than

‚Ä¢ > Greater than

‚Ä¢ <= Less than or equal to

‚Ä¢ >= Greater than or equal to

DETAILED EXPLANATION

int distance = 25;

// EQUAL TO (==)
if (distance == 25) {  // TRUE - exact match
  Serial.println("Exactly 25");
}

// NOT EQUAL (!=)
if (distance != 30) {  // TRUE - 25 is not 30
  Serial.println("Not 30");
}

// LESS THAN (<)
if (distance < 30) {  // TRUE - 25 is less than 30
  Serial.println("Less than 30");
}

// GREATER THAN (>)
if (distance > 20) {  // TRUE - 25 is greater than 20
  Serial.println("Greater than 20");
}

// LESS THAN OR EQUAL (<=)
if (distance <= 25) {  // TRUE - 25 equals 25
  Serial.println("25 or less");
}

// GREATER THAN OR EQUAL (>=)
if (distance >= 10) {  // TRUE - 25 is greater than 10
  Serial.println("10 or more");
}

Testing Ranges:

int temp = 25;

// Check if temperature is between 20 and 30
if (temp >= 20 && temp <= 30) {
  Serial.println("Temperature in range");
}

// Check if value is outside range
if (temp < 20 || temp > 30) {
  Serial.println("Temperature out of range");
}
